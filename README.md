## FAQ：AI 角色协作与项目管理

### Q1: 如果我把项目同步到 GitHub 并在另一台电脑（如 Mac）克隆下来，还需要重新训练“小吏”吗？
**A: 不需要。**
AI 角色的记忆和智慧是**文件化**的。
- 小吏的所有感悟都保存在 `role/xiaoli/knowledge/` 下的 `.md` 文件中。
- 角色定义文件 `role/xiaoli/xiaoli.role.md` 引用了这些知识文件。
- 当您克隆项目到新电脑时，这些文件会原样复制过去。
- 您只需要在新环境**激活小吏**，他就会自动读取这些本地文件，立刻恢复到当前的“满级”状态，无缝衔接之前的进度。

### Q2: “小吏”能读取根目录下的文件（如 `cuotiji/`）吗？为什么？
**A: 能读取，但机制不同。**
- **能读取（工具能力）**：作为运行在 IDE 中的 AI，小吏拥有读取整个项目文件夹（Workspace）内任何文件的权限和能力。
- **是知识（角色设定）**：如果文件被 `xiaoli.role.md` 显式引用（如 `@!path/to/file`），它就是角色的“默认内存”；如果未被引用（如根目录的错题集），它就是“外部资料”。小吏可以主动去查阅外部资料，但不会默认将其作为核心知识时刻加载。
- **结论**：项目级的通用文档（如 README、错题集）放在根目录是合适的，小吏随时可以查阅。

### Q3: 在新环境中具体该如何激活“小吏”？
**A: “大喊一声”即可（输入指令）。**
在对话框中直接输入以下任意一种指令，AI 就会自动识别并调用工具加载角色：
- `激活小吏`
- `切换到小吏角色`
- `你现在是小吏`

AI 收到指令后，会去读取本地的 `role/xiaoli/xiaoli.role.md` 文件，瞬间“附体”并加载所有记忆。
